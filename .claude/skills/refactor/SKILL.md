# Skill：重構建議 (refactor)

## 用途
分析指定檔案，提供具體重構建議，提升可讀性與可維護性。

## 觸發
```
/refactor [檔案路徑]
```

## 執行流程
1. 讀取目標檔案（完整內容）
2. 識別重構機會：
   - 重複程式碼（DRY）
   - 過長函式（> 40 行考慮拆分）
   - 深層巢狀（> 3 層考慮提取）
   - 魔法數字（應定義為常數）
   - 命名不清的變數 / 函式
   - React 元件過大（考慮拆分子元件）
3. 提供 before / after 對比範例
4. 評估重構風險（低 / 中 / 高）

## 本專案常見重構機會
- controller 的重複 try/catch 模式可提取為 `withClient()` helper
- 前端表單欄位重複可提取為 `<Field>` 元件
- API 錯誤處理重複邏輯可提取為 `handleApiError()`

## 原則
- 不改變外部行為，只改善內部結構
- 小步重構，每次只做一件事
- 重構前確認現有行為正確
